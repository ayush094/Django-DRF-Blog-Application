┌─────────────────────────────────────────────────────────────────────────────┐
│                        USER JOURNEY FLOW                                   │
└─────────────────────────────────────────────────────────────────────────────┘

  1. User → Visits /audio-hub/?blog_id=1
  2. UI → Loads blog info & existing audio list
  3. User → Selects mode/language/level/mood
  4. User → Clicks "Generate Audio"
  5. Frontend → POST /api/blogs/1/audio/generate/
  6. Backend → Creates AudioFile(status=pending)
  7. Celery → Receives task via generate_blog_audio_task
  8. AI Service → Claude transforms content + TTS generates audio
  9. Backend → Updates AudioFile(status=ready, audio_file=path)
  10. MQTT → Sends notification (optional)
  11. Frontend → Auto-refreshes player or status badge
  12. User → Clicks Play → Audio streams from MEDIA_URL
  13. User → Adds bookmarks, adjusts preferences, takes quiz
  14. All user data → Saved to database via respective APIs
